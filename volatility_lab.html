⚠ Common Mistakes
Disk image before RAM — The most common wrong answer. Students think "save the files" but RAM holds running processes, encryption keys, and network state that vanish on shutdown.
Swap / page file confusion — Swap is more volatile than disk but LESS volatile than RAM. It's a memory overflow area on disk — collect RAM first, swap second.
Network state overlooked — Active connections, ARP cache, and routing tables are in RAM and disappear immediately. Capture netstat and ARP before anything.
Logs are NOT volatile — System event logs persist on disk. They are collected near-last, not first.
Shutting down first — Powering off to "preserve" the disk destroys all RAM evidence. Never shut down a live system before capturing volatile data.
